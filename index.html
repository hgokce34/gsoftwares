Web Geliştirme Mühendisliği
Web Development Engineering

Kod yazmıyoruz;
sistem inşa ediyoruz. We don't write code;
we architect systems.

Bir web uygulaması, yalnızca görsel bir arayüzden ibaret değildir. Arka planda çalışan veritabanı mimarisi, gerçek zamanlı senkronizasyon katmanı, güvenlik kuralları ve ölçeklenebilirlik planı — bunlar, yazılımın görünmez iskeletidir. Biz bu iskeleti inşa etmekten başlarız. A web application is not merely a visual interface. The database architecture running in the background, the real-time synchronization layer, security rules and scalability plan — these are the invisible skeleton of software. We start by building this skeleton.

Serverless mimariler, Firebase Firestore gerçek zamanlı dinleyicileri, Cloudinary medya optimizasyonu ve SheetJS ile Excel entegrasyonu — bunların tamamı, müşterimizin günlük iş akışına dokunacak şekilde tasarlanır. Teknoloji seçimi hype'a değil, iş gereksinimine dayanır. Serverless architectures, Firebase Firestore real-time listeners, Cloudinary media optimization and Excel integration via SheetJS — all of these are designed to touch our client's daily workflow. Technology selection is based on business requirements, not hype.

Mimari Kararlar

Architectural Decisions

NEDEN TEK DOSYA?
WHY SINGLE FILE?

Deployment karmaşıklığını sıfıra indirmek için. GitHub Pages üzerinde anında yayın, CDN gereksizliği ve bakım kolaylığı.

To reduce deployment complexity to zero. Instant publishing on GitHub Pages, no CDN requirement and ease of maintenance.

NEDEN FIREBASE?
WHY FIREBASE?

Kendi backend'ini yazmak zaman ve maliyet demektir. Firebase Firestore ile gerçek zamanlı senkronizasyon, Auth ile kullanıcı yönetimi ve güvenlik kuralları — tümü yönetilen servis olarak.

Writing your own backend means time and cost. Real-time synchronization with Firebase Firestore, user management and security rules with Auth — all as managed services.

NEDEN VANILLA JS?
WHY VANILLA JS?

Bu ölçekteki bir uygulama için React'ın virtual DOM overhead'i gereksizdir. ES2024 modules, async/await ve native DOM API'leri ile aynı güç, sıfır bağımlılık.

For an application of this scale, React's virtual DOM overhead is unnecessary. Same power with ES2024 modules, async/await and native DOM APIs, zero dependencies.

İŞ KANUNU HESAPLAMA
LABOR LAW CALCULATION

Türk İş Kanunu 53. Madde — 1-5 yıl: 14 gün, 5-15 yıl: 20 gün, 15+ yıl: 26 gün, 50+ yaş: minimum 20 gün — saf JavaScript ile, harici API olmadan hesaplanır.

Turkish Labor Law Article 53 — 1-5 years: 14 days, 5-15 years: 20 days, 15+ years: 26 days, 50+ age: minimum 20 days — calculated in pure JavaScript, without external API.
